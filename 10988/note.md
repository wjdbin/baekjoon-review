# 📝 C언어 오답노트: 백준 10988번 (팰린드롬 확인하기)

이 문서는 백준 10988번 문제를 C언어로 해결하는 과정에서 발생한 오류와 질문들을 정리한 오답노트입니다.

---

## 1. 문제
**백준 10988번: 팰린드롬인지 확인하기**
* **내용:** 알파벳 소문자로만 이루어진 단어가 주어질 때, 이 단어가 팰린드롬인지 아닌지 확인하는 문제.
* **팰린드롬:** 앞으로 읽을 때와 거꾸로 읽을 때 똑같은 단어 (예: level, noon)
* **출력:** 팰린드롬이면 1, 아니면 0을 출력.

---

## 2. 내가 겪은 오류 및 질문

### 오류 1: `scanf`로 문자열 입력받기

C언어에서 문자열(단어)을 입력받기 위해 `char word[101];` 배열을 선언했습니다.

* **잘못된 코드:**
    ```c
    scanf("%d", word);
    ```
* **문제점:**
    * `%d`는 **정수(integer)**를 입력받기 위한 형식 지정자입니다.
    * `word`는 **문자 배열(char array)**이므로, 문자열을 입력받기 위해서는 **`%s`**를 사용해야 합니다.
    * `%d`를 사용하면 프로그램이 비정상 종료되거나 완전히 잘못된 값을 읽어옵니다.
* **올바른 코드:**
    ```c
    scanf("%s", word);
    ```

### 오류 2: 팰린드롬 판별 로직 (출력 위치)

문자열의 양쪽 끝을 비교하는 `for` 반복문 로직에서 출력을 잘못된 위치에 두었습니다.

* **잘못된 코드:**
    ```c
    int len = strlen(word);

    for (int i = 0; i < len / 2; i++)
    {
        if (word[i] != word[len - 1 - i])
        {
            printf("0");
            return 0; // 다른 것을 찾았으니 0 출력 후 종료 (이 부분은 맞음)
        }
        else
        {
            // 🚨 문제의 원인
            printf("1"); // 같을 때마다 1을 출력함
        }
    }
    ```
* **문제점:**
    * 위 코드는 `else` 블록 때문에 문자가 일치할 때마다 `1`을 출력합니다.
    * 예를 들어 "level"을 입력하면, 'l'과 'l'이 같아서 `1`을 출력하고, 'e'와 'e'가 같아서 `1`을 또 출력합니다.
    * **결과:** "11"이 출력됩니다.
* **올바른 로직:**
    * 팰린드롬 판별은 "단 한 번이라도 다르면 '실패(0)'"이고, "반복문이 끝날 때까지 다른 적이 한 번도 없으면 '성공(1)'"입니다.
    * 따라서 `1`은 **반복문이 모두 끝난 후에** 출력해야 합니다.

* **올바른 코드:**
    ```c
    int len = strlen(word);

    for (int i = 0; i < len / 2; i++)
    {
        // 1. 다른 경우를 찾으면 0을 출력하고 즉시 종료
        if (word[i] != word[len - 1 - i])
        {
            printf("0");
            return 0;
        }
        
        // 2. else 블록 자체를 삭제 (같을 때는 아무것도 안 하고 다음 검사로 넘어감)
    }

    // 3. 반복문이 중간에 종료되지 않고 무사히 끝났다면, 팰린드롬임
    printf("1");
    ```

### 질문 3: Visual Studio의 `scanf` 경고

* **질문:** "visual studio 에서 scanf 쓰지만 하면 반환값이 무시됐다고 할까"
* **답변:**
    * `scanf` 함수는 **성공적으로 입력받은 항목의 개수**를 정수(int)로 반환(return)합니다. (예: `scanf("%s", word);`가 성공하면 1 반환)
    * Visual Studio의 컴파일러는 이 **반환값**을 우리가 확인하지 않았다고 경고를 띄우는 것입니다. ("`scanf`가 성공했는지 실패했는지 확인 안 했네요?")
    * 이 경고는 문법 "오류(Error)"가 아닌 "경고(Warning)"입니다.
    * 알고리즘 문제 풀이(백준 채점) 시에는 이 경고를 무시해도 괜찮습니다. (백준에서 사용하는 GCC 컴파일러는 이 경고를 기본으로 띄우지 않습니다.)
    * 더 안전한 코드를 위해 `scanf_s`를 사용하라고 권장하기도 하지만, 백준에서는 `scanf_s`를 지원하지 않으므로 `scanf`를 사용하는 것이 맞습니다.
