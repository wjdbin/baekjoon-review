# 📝 백준 2941 (크로아티아 알파벳) 오답노트

## 1. 내가 겪은 문제 (The Mistake)

### ❌ 치명적인 오류: `scanf` 사용법과 `strlen` 호출 순서

가장 기본적인 입력과 문자열 길이 계산에서 실수가 있었다.

1.  **`scanf` 오류:** 문자열을 저장할 배열 `str`이 아닌, 정수형 변수 `len`에 입력을 받으려고 했다.
    * **(틀린 코드)** `scanf("%s", len);`
    * **(수정)** `scanf("%s", str);` // `%s`는 문자열이므로, `char` 배열의 이름(주소)인 `str`을 넘겨야 한다.

2.  **`strlen` 순서 오류:** `scanf`로 `str`에 문자열을 **입력받기도 전에** `strlen(str)`을 호출했다.
    * **(틀린 순서)**
        ```c
        char str[101];
        int len;
        scanf("%s", len);     // 1. str이 아닌 len에 받음
        len = strlen(str);    // 2. str은 비어있음 (쓰레기 값)
        ```
    * **(올바른 순서)**
        ```c
        char str[101];
        int len;
        scanf("%s", str);     // 1. str 배열에 먼저 문자열을 받는다.
        len = strlen(str);    // 2. 입력받은 str의 길이를 계산한다.
        ```

---

## 2. 문제 해결의 핵심 (Key Takeaways)

### 💡 핵심 1: `strlen`을 쓰려면 `string.h`가 필요하다.

* `strlen()` 함수는 C 표준 라이브러리의 **"string.h"** 헤더 파일에 정의되어 있다.
* 문자열의 길이를 계산하거나, 문자열 복사(`strcpy`), 비교(`strcmp`) 등 문자열 관련 함수를 사용하려면 반드시 `#include <string.h>`를 선언해야 한다.
* **적용:** 코드 최상단에 `#include <stdio.h>`와 함께 `#include <string.h>`를 추가했다.

### 💡 핵심 2: "첫 글자 확인, 다음 칸 공간 확인" (경계 검사)

이 문제의 로직은 **"현재 인덱스 `i`를 기준으로 다음( `i+1` ) 또는 다다음( `i+2` ) 칸을 안전하게 확인하는 것"**이다.

* **"첫 글자 확인":** `for` 문 안에서 `if(str[i] == 'c')`처럼 현재 문자를 확인한다.
* **"다음 칸 공간 확인":** `str[i+1]`을 확인하기 **전에**, `i+1`이 전체 길이 `len`보다 작은지(`i + 1 < len`) **반드시 먼저 확인**해야 한다.
    * 이 '경계 검사(Boundary Check)'가 없으면, `i`가 마지막 인덱스일 때 `str[i+1]` (존재하지 않는 메모리)에 접근하여 **런타임 에러(Segmentation fault)**가 발생한다.
* **"다다음 칸 공간 확인":** `dz=`를 처리할 때 `str[i+2]`를 확인해야 하므로, `i + 2 < len`인지 확인해야 한다.

#### ✅ 올바른 로직 적용 (C 코드 예시)

```c
// 'd'로 시작하는 경우
else if(str[i] == 'd' && i + 1 < len) // 1. 첫 글자 'd' 확인 + 2. "다음 칸 공간" 확인
{
    // 'd-' 확인
    if(str[i + 1] == '-')
    {
        i++;
    }
    // 'dz=' 확인
    // 3. "다다음 칸 공간" 확인
    else if(str[i + 1] == 'z' && i + 2 < len) 
    {
        if(str[i + 2] == '=')
        {
            i += 2; // 'dz='는 3글자이므로 2칸 건너뛰기
        }
    }
}
